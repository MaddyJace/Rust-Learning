# 认识所有权
所有权是`Rust`的特性，通过所有权可以保证`Rust`的内存安全，因此理解所有权在`Rust`中是如何工作的非常重要。\
在本章中，我们将了解所有权：拷贝， 借用（borrowing）、slice，以及 Rust 如何在内存中布局数据 [官方说明](https://doc.rust-lang.org/book/ch04-00-understanding-ownership.html)。

---

### 所有权规则
1. Rust 中的每一个值都有一个 所有者（owner）。
2. 值在任一时刻有且只有一个所有者。
3. 当所有者离开作用域，这个值将被丢弃。
4. 如果你把一个变量赋值给另一个变量，所有权会发生移动(Move)，原变量将不再可用。这避免了“双重释放”内存的风险。

---

### 变量作用域
```rust
fn main() {
    {                      // s 在这里无效，它尚未声明
        let s = "hello";   // 从此处起，s 是有效的

        // 使用 s
    }                      // 此作用域已结束，s 不再有效
}
```

---

### 所有权转移
值在作用域内有自己的所有权，重新赋给新变量所有权会被转移给新变量，旧的变量名不能再使用。
```rust
fn main() {
    let s1 = String::from("hello");
    let s2 = s1; // s1所有权已经给了s2，s1不能再使用

    println!("{s1}, world!"); // 编译器报错
}
```

---

### 变量拷贝
如果将一个已知大小的变量赋值给新的变量会将旧变量拷贝给新变量(`完全独立互不干扰`)，直到作用域结束被丢弃。
> 参阅: [已知大小类型](数据类型.md): `i8, i16, i32, i64, i128, isize, u8, u16, u32, u64, u128, usize, f32, f64` 。
```rust
fn main() {
    let x = 5;
    let y = x; // 这个大小类型x被复制到了y

    println!("x = {x}, y = {y}"); // x和y都是有效的
}
```
> [!CAUTION]
> 对于存放在堆上的复杂数据对象`(如 String)`不会进行拷贝，重新指向新变量会发生所有权转移，即意味着旧变量无法再访问。\
> 如果你想将旧的`String`变量拷贝一份给新的`String`变量，需要显式调用`String`类型提供的`.clone()`方法，其它类型需自己实现。

---

### 引用
如果你不想拥有变量的所有权又想访问它的值，可以使用引用它在变量名的开头加上`&`符号表示引用。
```rust
fn main() {
    let s1 = String::from("hello");
    let len = calculate_length(&s1);
    println!("The length of '{s1}' is {len}.");
}

fn calculate_length(s: &String) -> usize {
    s.len()
}
```
> 在这个例子中`calculate_length`函数它去s的字符串长度，最终`s1`和`len`依然可以使用。
