# Rust 数据类型转换
在 Rust 中，类型转换是**显式**且**严格**的。为了编写安全的代码，我们需要从三个不同的“视角”来处理类型转换。

---

### 1. 强制转换: `as`关键字
Rust 中最基础的转换方式，主要用于原始数值类型`整数、浮点数`之间的互相转换。
* **适用场景**：数值类型互转、指针转换。
* **注意**：`as` 转换是**有损**的部分数据会丢。如果将大类型转为小类型(如`i32`转`u8`)会发生**位截断**，非必要场景尽量少用。

```rust
fn main() {
    let decimal: f64 = 65.9876;

    // f64 -> i32 (小数部分被直接切除)
    let integer = decimal as i32; 
    println!("原浮点数: {}, 转换后整数: {}", decimal, integer); // 65

    // i32 -> u8 (如果值超过 255，会发生位截断)
    let char_code = 65_i32;
    let character = char_code as u8 as char;
    println!("数值 {} 对应的字符是: {}", char_code, character); // 'A'
}
```

### 2. 无损转换视角：`From`与`Into`
Rust 标准库中最推崇的转换方式。它遵循“只要转换是绝对安全的，就提供实现”的原则。
* **适用场景**：低精度转高精度（如`i32`转`i64`）、类型包装（如`&str`转`String`）。
* **特点**：**绝对安全**，不会丢失数据。
```rust
fn main() {
    // 场景 A: 低精度转高精度 (自动实现)
    let small_num: u32 = 100;
    let big_num: u64 = u64::from(small_num); // 使用 From
    let another_big: u64 = small_num.into(); // 使用 Into (需指定目标类型)

    // 场景 B: 字符串转换
    let slice: &str = "Hello Rust";
    let string: String = String::from(slice);
    
    println!("无损转换成功: {} 和 {}", big_num, string);
}
```

### 3. 解析转换
当你需要把字符串（String/&str）转换为数值类型时，必须使用解析转换。
* **适用场景**：处理用户输入、读取配置文件。
* **特点**：可能失败。因为字符串内容不确定`parse`返回的是一个`Result`枚举，必须处理潜在的错误。
```rust
fn main() {
    let price_str = "125";
    let age_str = "未知";

    // 正确解析：使用 expect 或 unwrap 处理 Result
    let price: i32 = price_str.parse().expect("无法解析价格");

    // 错误处理：使用 match 安全处理异常
    let age = age_str.parse::<i32>();
    match age {
        Ok(num) => println!("解析成功: {}", num),
        Err(_) => println!("解析失败：'{}' 不是一个有效的数字", age_str),
    }

    println!("最终价格: {}", price);
}
```

### 4. 总结转换对照表
| 视角 | 方式 | 安全性 | 核心说明 |  `                                                  ` |
| :--- | :--- | :--- | :--- | :--- |
| 强行转换 | `as` | 🔴 低 | 简单粗暴，小心截断和精度丢失。 | `                                                  ` |
| 无损转换 | `From`/`Into` | 🟢 高 | 编译器保证转换过程不丢数据。 | `                                                  ` |
| 解析转换 | `parse()` | 🟡 中 | 专门用于字符串转数值，必须处理错误。 | `                                                  ` |
